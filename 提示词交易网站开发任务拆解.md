# 提示词交易网站开发任务拆解

更新时间：2026-04-30

## 1. 当前项目状态

已完成：

- 静态首页。
- 本地提示词数据接入。
- 图片与提示词卡片展示。
- 前 10 条免费复制。
- 9.9 元会员模拟解锁。
- 搜索和语言筛选。
- 需求调研文档。

当前限制：

- 没有真实登录。
- 没有真实支付。
- 会员状态只保存在浏览器本地。
- 完整提示词数据在前端文件里，不能用于真实收费。
- 没有后台管理。

## 2. 技术路线建议

### 2.1 快速上线路线

适合目标：

- 尽快上线验证付费。
- 内容由管理员维护。
- 暂时不开放创作者。

建议技术栈：

- 前端：Next.js 或现有静态站继续演进。
- 后端：Next.js API Routes 或 Node.js Express。
- 数据库：PostgreSQL。
- 文件存储：S3/R2/阿里云 OSS。
- 支付：微信支付或支付宝。
- 部署：Vercel/服务器/宝塔均可。

### 2.2 国内部署路线

适合目标：

- 面向国内用户。
- 接微信支付和支付宝。
- 需要备案域名。

建议：

- 前端和后端部署到国内云服务器。
- 图片放阿里云 OSS 或腾讯云 COS。
- 数据库使用云 PostgreSQL 或 MySQL。
- 支付优先接微信 Native 支付或 JSAPI 支付。

## 3. 第一阶段：前端产品原型

任务：

- 重构首页为内容优先布局。
- 增加分类导航。
- 增加提示词详情弹窗。
- 增加免费/会员状态标签。
- 增加收藏按钮 UI。
- 增加复制成功反馈。
- 增加移动端适配。

验收：

- 首屏能看到真实提示词内容。
- 点击卡片能看到详情。
- 免费内容可复制。
- 会员内容会触发购买弹窗。
- 手机宽度 375px 下无内容溢出。

## 4. 第二阶段：账号系统

任务：

- 注册。
- 登录。
- 退出登录。
- 用户信息接口。
- 登录态持久化。
- 受保护页面。

建议实现：

- 手机号验证码登录，适合国内产品。
- MVP 也可以先用邮箱密码登录。

验收：

- 未登录用户点击收藏时跳登录。
- 登录后能进入“我的”页面。
- 用户信息刷新后仍保持登录。

## 5. 第三阶段：服务端权限

任务：

- 将完整提示词从前端移到服务端。
- 列表接口只返回摘要。
- 详情接口按权限返回完整提示词。
- 复制接口按权限返回可复制正文。
- 复制行为写入日志。

验收：

- 非会员无法从前端源码拿到全部提示词。
- 第 11 条之后必须走会员校验。
- 会员可以复制全部提示词。

## 6. 第四阶段：支付与会员

任务：

- 创建会员商品。
- 创建订单。
- 接支付平台。
- 支付回调。
- 订单状态查询。
- 支付成功开通会员。

验收：

- 用户可支付 9.9 元。
- 支付成功后立即解锁。
- 刷新页面后会员状态仍然有效。
- 服务端能查询订单记录。

## 7. 第五阶段：后台管理

任务：

- 管理员登录。
- 提示词列表。
- 新增提示词。
- 编辑提示词。
- 上传图片。
- 上架/下架。
- 分类管理。
- 订单管理。
- 用户管理。

验收：

- 管理员能独立维护内容。
- 下架内容前台不可见。
- 后台可查支付订单。

## 8. 第六阶段：运营增长

任务：

- SEO 标题和描述。
- 分类页静态化。
- 热门榜单。
- 新增内容提醒。
- 会员转化数据。
- 复制转化漏斗。

指标：

- 首页访问量。
- 提示词详情点击率。
- 免费复制率。
- 会员弹窗打开率。
- 支付转化率。
- 复购/回访率。

## 9. 推荐开发顺序

建议按下面顺序推进：

1. 把静态页面改成内容优先的交易站原型。
2. 增加详情弹窗，提升提示词展示质量。
3. 增加分类和权限标签。
4. 接服务端接口，隐藏完整付费提示词。
5. 做登录和会员状态。
6. 接支付。
7. 做后台。

原因：

- 先验证用户是否对内容感兴趣。
- 再做权限和支付，避免过早投入后端。
- 最后做后台和创作者，减少第一版复杂度。

## 10. 待确认问题

上线前需要确认：

- 会员 9.9 元是永久会员还是月会员。
- 第三方来源提示词是否允许商业售卖。
- 是否只做 GPT Image，还是支持 Midjourney、SD、Seedream。
- 支付优先接微信还是支付宝。
- 是否需要手机号登录。
- 后台是否由一个管理员使用，还是多人协作。
